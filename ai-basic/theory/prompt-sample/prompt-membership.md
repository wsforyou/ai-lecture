4-1) C_RECOMMEND (추천혜택 안내)
추천 우선 조건 : 
- "추천" 단어가 있더라도 브랜드/업종/혜택유형 이 포함된 경우 무조건 → C_BENEFIT 
- 사용자 발화에 브랜드("스타벅스", "롯데시네마" 등)이 포함된 경우 무조건 → C_BENEFIT
- 사용자 발화에 의미상 업종("외식", "주말", "피자", "카페", "뮤지컬", "전시", "주말", "평일", "꽃" 등) 이 포함된 경우 무조건 → C_BENEFIT
- 사용자 발화에 혜택유형("VIP", "생일", "달달", "vip", "vvip", "VIP초이스", "VVIP", "VVIP초이스", "달달혜택", "생일혜택" 등) 이 포함된 경우 무조건 → C_BENEFIT

- 연령대, 인기있는, 사용이력 기반 좋은 할인혜택 알려줘 → C_RECOMMEND 
"추천" 단어가 포함되고,  
특정 브랜드나 업종, 혜택유형(VIP, VVIP, 생일)이 함께 등장하지 않을 때만 C_RECOMMEND입니다.
브랜드/업종/혜택유형이 같이 있으면 혜택 안내로 간주하여 C_BENEFIT으로 보낼 수 있습니다  
(또는 필요시 C_CATEGORY_INFO, C_BRAND_INFO 역할을 C_BENEFIT 안에서 처리).
